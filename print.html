mdbook プロジェクトを作成

cd Desktop

mkdir mdbook_test

cd mdbook_test

リポジトリに登録するための SSH 鍵を作成する(mdbook_test にファイルで保存される)※あとで消す

ssh-keygen -t rsa -b 4096 -f gh-pages
or
ssh-keygen -t rsa -b 4096 -C "$(git config user.email)" -f gh-pages -N ""

github レポジトリを２つ作成。 1 つは mdbook を作成する プロジェクト 用, 1 つは GitHub Pages 用

[mdbook を作成する プロジェクト 用の作業]

  • Settings > Secrets に任意の名前で登録します(今回の名前は"ACTIONS_DEPLOY_KEY")

  • key に 秘密鍵(gh-pages)をペースト

  • Actions > new workflow > Simple workflow > start commit

[GitHub Pages 用]

  • Settings > Deploy keys に"ACTIONS_DEPLOY_KEY"という名前で登録します

  • key に(gh-pages.pub)をペースト

  • Allow write access にチェックを入れる(このキーでこのリポジトリにプッシュできますか？デプロイキーは常にプルアクセスが可能です。)

mdbook_test の作業

  • mdbook_test に保存されている(gh-pages.pub)と(gh-pages)は使い回しできないので削除して良い

  • 設定ファイルの２つ作成

".github/actions/build/Dockerfile"

FROM rust:latest

RUN cargo install mdbook --no-default-features --features output --vers "^0.3.5"

CMD ["mdbook", "build"]

".github/workflows/main.yml"


name: CI/CD
on:
  push:
    branches:
    - master
    paths:
    - "src/**"
    - "book.toml"

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v2
    - name: Build
      uses: ./.github/actions/build
    - name: Deploy
      uses: peaceiris/actions-gh-pages@v3
      with:
        deploy_key: ${{ secrets.ACTIONS_DEPLOY_KEY }}
        EXTERNAL_REPOSITORY: abe-masafumi/mdbook_Pages_about_mdbook
        PUBLISH_BRANCH: master
        PUBLISH_DIR: ./book

コマンド一覧

表示（listの略）

「ls」+「オプション」+「ファイル、ディレクトリ名」

ls  #カレントディレクトリにあるファイルやディレクトリを表示する
ls -l  #ファイルやディレクトリの詳細も同時に表示する
ls -a  #隠しファイルも含めファイルやディレクトリを全て表示する
ls −la  #隠しファイルを含む全てのファイルやディレクトリの詳細を表示する
pwd コマンド #カレントディレクトリのパスを表示する。（print working directoryの略）

オプション	機能
-a	        隠しファイル含めファイル全表示
-A	        「.」と「..」以外の隠しファイル含め全表示
-l	        詳細で表示
-l -@	        拡張属性の内容を表示
-l -e	        ACL設定がされているディレクトリを表示
-l -h	        サイズを適正な単位で表示
-l -u	        最終アクセス日を表示
-l -U	        作成された日時を表示
-F	        名前の末尾に種類を意味する記号を表示
-t	        日時を最近のものから並び替えて表示
-R	        サブディレクトリの内容も表示
-S	        サイズの大きいものから並び替えて表示
-r	        名前を降順に表示
-d	        指定したディレクトリの情報を表示
-o	        グループ名を省略して詳細表示
-g	        所有者名を省略して詳細表示
-G	        種類に応じて色を変更して表示

移動（change directoryの略）

cd test.html  # testテストというファイルに移動する
cd ~/  #ホームディレクトリに移動する
cd ..  #一つ上の階層のファイルに移動する

ファイルの作成（print working directoryの略）

touch コマンド #中身が空のファイルを作る。
touch test.html  #test.htmlという中身が空のファイルを作る

ディレクトリの作成（make directoryの略）

mkdir test #testというファイルを作る

ファイルの移動、名前変更、コピー（moveの略）（copyの略）

mv test.html tmp/  #test.htmlというファイルを相対パスでtmp/に移動する
mv test.html test2.html  #test.htmlというファイルをtest2.htmlに名前変更する
cp コマンド #ファイルをコピーする。
cp test.html tmp/  #test.htmlを相対パスでtmp/というディレクトリの中にコピーする
cp test.html test2.html  #test.htmlをtest2.htmlという名前でコピーする
cp −r dir /tmp/  #dirというディレクトリとその中身を絶対パスで/tmp/にそっくりコピーする

ファイルの編集

vi ファイル名 #ファイルを指定して開始

i #カーソルの位置で入力モードに切り替え

esc #コマンドモードに切り替え

:w #保存

:w ファイル名 #名前をつけて保存（新しくファイルをつくる）

:q #終了

:wq #保存して終了

削除（removeの略）

rm test.html  #test.htmlを削除する
rm -r test  #testというディレクトリとその中身を削除する
rm -f test  #testというファイルを警告なしで削除する
rm -rf test  #testというディレクトリとその中身を警告なしで削除する

開く

open .  #現在のターミナルのディレクトリFinderで開く
open ~/  #ホームディレトリをFinderで開く

読み込み有効

source ~/.bash_profile  #ホームディレクトリにある.bash_profileに書き込んだ設定を読み込み有効にする
source .bash_profile  #カレントディレクトリにある.bash_profile設定を読み込み有効にする

履歴

history  #これまで実行してきたコマンドの履歴を表示する
history -c  #これまで実行してきたコマンドの履歴を消去する

差分を表示

diff [ファイル1] [ファイル2]
